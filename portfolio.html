Genetic Algorithms for Maximizing Investment Portfolios

If you haven't already looked over the first example, SAGA, it would be a good idea to look it over as the following example does not lend itself to images.

The following is an example of how genetic algorithms can be used to optimize investment portfolios.

Genetic algorithms are used for investment portfolios becuase they are able to take into account both major points of investing, risk and return. Previously to this most algorithms were linear or quadratic and were only able to take into account one of these as a variable, which was almost alway return. The problem with that is that not all investors are ok with high amounts of risk and will often sell to cut thier losses, rather than wait it out for the possibility of large returns later. Genetic algorithms not only allow both to be variables but can take into account any number of variables based on the portfolio contents. The end goal of such an algorithm is to find the portfolio that is able to maximize return while minimizing risk.

To start off with the initial population unlike the previous example is much larger and is broken up into sub-categories, called species. Each of these species is divided by the value of their trade-off coefficient, the value that determines how much to prioritize risk over return and vice versa. Each is then scored based on their projected return and risk. In this algorithm unlike the previous example 100% of the next generation is created using the objective functions, this is because a good genetic algorithm will naturally keep in better solutions (just like in evolution).

The objective functions for this example aim to create variation in the contents of the porfolios. To do this the crossover and mutation functions change not only which stocks from the selection are chosen but how much of each is purchased in the porfolio. Unfortunately this is where the example gets a little hazy. Because this is an algorithm used for investment the details as to how each objective function, or the score for that matter, actually works is not detailed in the published article. This is quite common in investment algorithm publication, people publish thier data but the actual details of their algorithm are vital to maximizing their own, or thier company's profit. Even without such details, objective functions all aim at maximizing variation in the data to increase likelyhood of arriving at the global optimal. The run time for this algorithm and most algorithms in the field is under 10ms; as slower algorithms might be susceptible to being manipulated by faster algorithms searching for slow algorithms.
